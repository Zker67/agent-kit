# 计划文档索引

> 单一入口：所有设计决策、实施计划、改进追踪都在 `plans/` 目录下管理。

## 何时创建计划

- 新项目启动或需要明确架构边界时。
- 重大重构、跨模块改动或高风险变更前。
- 新功能涉及多个阶段、多个文件或需要验收标准时。

## 文档结构约定

```text
plans/
├── README.md
├── YYYY-MM-DD-feature-name/
│   ├── 00-overview.md
│   ├── 01-phase-name.md
│   └── 09-improvements.md
└── archive/
```

## 每个阶段文档建议包含

1. 目标：本阶段要达成什么。
2. 不变量：哪些行为或接口不能破坏。
3. 实现清单：文件、接口、数据流和边界条件。
4. 验收标准：如何确认完成。
5. 回滚说明：失败时如何恢复。

## 状态标注

| 标注 | 含义 |
|---|---|
| `[done]` | 已完成并验证 |
| `[partial]` | 核心可用但有遗留问题 |
| `[planned]` | 已规划，未执行 |
| `[dropped]` | 已废弃或被替代 |
