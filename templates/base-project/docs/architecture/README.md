# 架构索引

> 本目录只做结构、边界和长期技术事实的分发，不重复保存计划过程或外部资料原文。

## 当前索引

- [单一信息源说明](./source-of-truth.md)
- [项目结构总览](./project-structure.md)

## 阅读顺序

1. 先看 [项目结构总览](./project-structure.md)，理解当前项目目录和模块边界。
2. 再看 [单一信息源说明](./source-of-truth.md)，确认同一类信息的主维护位置。
3. 新增专题文档时，在本索引登记入口。

## 维护原则

- 层层索引：上层只分发，不与下层争夺解释权。
- 单一信息源：同一类信息只在一个主位置完整维护。
- 信息一致性：README、AGENTS、docs、plans 和 references 之间只保留必要摘要和链接。
