# 单一信息源说明

> 本文件定义“哪类信息应该信哪个文件”。同一概念在多处出现时，只能有一个主维护位置。

## 总原则

- 主维护位置负责完整事实。
- 其他文件只做入口、摘要、状态或引用，不维护平行版本。
- 如果文档之间发生冲突，以本文件声明的主维护位置为准。

## 信息归属矩阵

| 信息类型 | 主维护位置 | 其他文件角色 |
|---|---|---|
| 项目定位、快速启动、文档分流 | `README.md` | 其他文档只引用 |
| AI 协作入口、开发约束、项目不变量 | `AGENTS.md` | `README.md` 只提示入口 |
| 目录结构、模块职责、架构边界 | `docs/architecture/project-structure.md` | README 和 AGENTS 只做摘要 |
| 单一信息源规则 | `docs/architecture/source-of-truth.md` | 其他文档只链接 |
| 运行、配置、部署 | `docs/operations.md` | README 只保留最短启动说明 |
| API、接口、schema、事件合同 | `docs/api.md` 或对应专题文档 | README 和计划只做入口 |
| 分阶段方案、设计草案、执行记录、验收状态 | `plans/README.md` 和 `plans/` 具体文件 | 完成后的稳定事实同步回 `docs/` |
| 外部文档、标准、教程、资料链接 | `references/external-docs/` | 当前项目事实应沉淀到 `docs/` |
| 外部代码库、参考实现、对比分析 | `references/external-repos/` | 采纳后的当前实现事实写入 `docs/` 或代码 |
| 项目级 AI 规则 | `.agent/rules/` | AGENTS 只说明入口 |
| 长期上下文和当前任务记忆 | `.ai_memory/` | 不替代 README、docs 或 plans |
| 可复用经验 | `.exp/` | 不承担当前项目事实源 |
| UI/UX 风格说明 | `.ui/` | README 只做入口 |
| 本地个人笔记 | 由 `MEMO.example.md` 复制出的本地文件 | 默认不作为项目事实源 |

## 外部参考规则

- `references/` 保存外部资料的来源、结构、借鉴点、版本或 commit、访问日期和许可边界。
- 外部资料不是当前项目事实源；如果外部资料影响了当前实现，应把最终决策写入 `docs/` 或源码注释。
- 不默认复制第三方代码、文档全文、构建产物或依赖管理状态；必要引用应保留链接和短摘要。

## 维护规则

1. 新增长期事实时，先判断应归属到哪个主维护位置。
2. 新增两个以上子文档的目录时，补 `README.md` 索引。
3. 新增、移动、归档计划时，同步更新 `plans/README.md`。
4. 新增重要外部资料时，登记到 `references/` 对应索引。
5. 只变入口时更新索引，不复制正文。
