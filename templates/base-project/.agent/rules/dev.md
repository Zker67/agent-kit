# 通用开发规则

## 语言

- 默认使用简体中文沟通、写文档和 UI 文案。
- 代码标识符、命令、文件路径、API 名称和错误信息保持原文。
- 注释解释意图、约束或取舍，不解释显而易见的语句。

## 工作流

- 修改前先理解现有结构、风格、测试和错误处理。
- 优先做根因修复，避免无关重构。
- 每处改动都应对应明确任务目标、缺陷修复或必要验证。
- 非平凡任务先写计划到 `plans/`，并同步维护 `plans/README.md`。

## 验证

- 改代码后按风险从小到大验证：定向测试、类型检查、lint、构建或 smoke test。
- 验证无法执行时，说明原因和下一个最合适检查。

## 安全

- 不提交真实凭据、机器路径、运行态数据、缓存或个人笔记。
- 处理外部输入时按风险校验路径、命令、文件、模板和序列化数据。
