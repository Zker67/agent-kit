# Pro Rule

在以下情况触发：
- 用户希望整理自己在当前工作区给过的提示指令。
- 用户希望把这些规则沉淀成可复用文档。
- 用户说 "把我的偏好记住 / 整理规则 / 记一下这个规则 / 下次别忘了"。

## 协同与互斥

- 被 `pro-summary` 在文档一致性审查时调用，也可被 `pro-memory` 在长期上下文整理时调用；用户单独调用时可独立运行。
- 与 `pro-exp` 区分：`pro-exp` 记录 **通用解决方案经验**，`pro-rule` 记录 **用户偏好 / 工作区硬性指令**。
- 与 `.windsurfrules` / `CLAUDE.md` 等 IDE 级规则文件区分：本 skill 只维护 `.agent/rules/`，不动 IDE 自带的规则文件（除非用户明确要求）。

## 执行要求

1. 总结当前工作区中用户已经表达过的稳定偏好与提示规则。
2. 在项目根目录下创建或更新 `.agent/rules/`。
3. **先列出 `.agent/rules/` 已有文档**，遇到同类规则要合并更新、同义合并，而不是新建；只有在规则主题明显不同时才新建文件。
4. 文档以语义清晰的 `指令名.md` 命名，避免重复和彼此冲突。
5. 不要把一次性的临时指令误写成长期规则；判断不了就问用户。
6. 若没有额外稳定规则，不要为了凑数而生成空规则文件。

## 输出要求

- 先给 `.agent/rules/` 现有规则清单。
- 再给本次 **新增 / 更新 / 合并 / 跳过** 的动作列表，每条附 "依据会话中的什么原话"。
- 最后给出最终文件路径，方便 IDE 直接跳转。
