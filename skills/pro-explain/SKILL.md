# Pro Explain

在以下情况触发：
- 用户表示自己是初学者，希望解释当前任务。
- 用户要求在代码中补充更详细的中文注释。
- 用户说 "这段代码什么意思 / 帮我讲讲 / 加注释 / 详细点解释"。

## 协同与互斥

- **默认只加注释，不动代码实现**；如果某段逻辑必须小幅重构才能讲清，先向用户说明并征求同意。
- 与 `pro-must` 并用时以 `pro-must` 为准（不得小幅重构）。
- 注释语言遵循 `use-chinese` 与用户全局规则：跟随文件既有注释语言，不强行翻译既有英文注释。

## 执行要求

1. 先用通俗中文解释当前任务在做什么、为什么这样做。
2. 再定位相关文件，补充必要的注释（按文件既有语言）。
3. 注释要解释意图、边界条件和关键逻辑，不要写废话注释（例如 `i++ // 自增`）。
4. 保持原有代码风格，不要为了注释而大幅改写实现。
5. 若某段逻辑确实过于晦涩、不做小幅重构无法讲清，先列出重构提案请用户确认。

## 输出要求

按三段结构回复：

- **一、任务在做什么**：通俗解释 + 关键术语（术语保留英文并附中文说明）。
- **二、修改清单**：文件路径 + 行号范围 + 注释要点摘要。
- **三、如需重构**（可选）：当前问题 / 重构提案 / 影响面 / 请用户决定是否执行。
