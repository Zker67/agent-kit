# Pro Explain

在以下情况触发：
- 用户表示自己是初学者，希望解释当前任务。
- 用户希望理解代码、报错、执行流程、架构关系或技术概念。
- 用户说 "这段代码什么意思 / 帮我讲讲 / 为什么这样写 / 详细点解释"。

## 协同与互斥

- **始终只读**：允许读取源码、配置、测试和日志作为解释依据，但不添加注释、不修改实现、不创建文件。
- 用户明确要求修改代码或补充注释时，该修改不属于本 skill；按用户请求切换到对应的实现流程。
- 与 `use-chinese` 并存：解释默认使用简体中文，代码标识符、命令和外部专名保持原文。

## 执行要求

1. 读取与问题直接相关的代码、配置、测试或错误信息，先确认实际行为。
2. 先用一句话说明整体目标，再按执行顺序解释关键流程。
3. 对术语、抽象层和隐含前提使用通俗中文说明，并保留原始英文名称。
4. 区分源码能够直接证明的事实、合理推断和仍缺少的证据。
5. 只解释理解当前问题所需的内容，不扩展成无关教程或重构建议。

## 输出要求

按三段结构回复；简单问题可以合并精简：

- **一、一句话结论**：这段代码、任务或报错的核心含义。
- **二、执行流程**：按实际顺序解释关键步骤、数据流或调用关系。
- **三、关键概念与限制**：解释必要术语、边界条件、风险或尚未确认的信息。
