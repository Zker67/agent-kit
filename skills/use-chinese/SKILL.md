# Use Chinese

在以下情况触发：
- 用户要求全部使用中文。
- 当前输出、文档、注释或 UI 文案出现了非中文内容且用户要求统一语言。
- 用户说 "说中文 / 中文回复 / 翻译成中文 / 全用简体"。

## 执行要求

1. 所有回复使用简体中文（包括计划、任务列表、改动说明、文档）。
2. **代码标识符、CLI flag、文件路径、API 名称、错误信息保留英文原文**，不要翻译，避免破坏可执行性。
3. **代码注释跟随文件既有注释语言**：
   - 文件本身以中文注释为主：新增 / 修改注释写中文。
   - 文件本身以英文注释为主：新增 / 修改注释保持英文，不强行翻译。
   - 文件无既有约定时，按项目主导语言决定；默认中文。
4. UI 文案默认使用简体中文；除非项目既有约定（如 i18n 键值或英文 UI）要求其他语言。
5. 检查相关文档；若内容不是中文但项目是中文项目，按需要翻译为简体中文，保留技术关键词英文。
6. 思考链可以用英文，但最终输出给用户的内容必须是简体中文。

## 与其他 skill 的关系

- 本 skill 是常驻控制位，不是触发具体动作；与其它 `pro-*` 动作型 skill 叠加使用。
- 与用户全局规则 (`Think in English, reply in 简体中文`) 完全对齐，本 skill 是该规则的项目级显式确认。
