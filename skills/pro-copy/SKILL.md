# Pro Copy

在以下情况触发：
- 用户想知道某个功能是否已有类似项目。
- 用户要求参考、借鉴、复用现有开源实现。
- 用户说 "有没有类似的项目 / 看看别人怎么做 / 借鉴一下 / 找个开源的"。

## 协同与互斥

- 联网检索走 `use-internet`：技术库/框架优先 Context7，其它走 smart-search-cli。
- 与 `pro-must` 冲突时以 `pro-must` 为准（不主动换用户已定路线）。
- 发现的候选可以喂给 `pro-idea` 做扩展方案评审，但不替代它。

## 执行要求

1. 用英文搜索 GitHub 及相关资料，优先找与当前功能最接近的项目。
2. 在相关性、活跃度和 star 数之间做平衡，默认优先阅读最相关且 star 较高的项目。
3. 阅读关键代码与目录结构，提炼可复用思路、接口设计和实现细节。
4. 需要本地复用时，只迁移必要部分，不要盲目整仓复制。
5. 注意 license、依赖规模和 blast radius（影响面）。

## 输出要求

以表格列出候选项目，至少包含：

| 项目 | URL | Stars | License | 相关性 | 可复用部分 | 需要适配 | blast radius |
|------|-----|-------|---------|--------|-----------|---------|--------------|

表格下方：

- **推荐首选**：1 个最贴合当前需求的项目 + 选它的理由。
- **可复用片段**：具体文件/函数/接口，说明为什么可复用。
- **风险提示**：license 冲突、依赖规模、维护活跃度、架构差异。
